System Overview

Single-tenant UAT webapp wrapping 4 Claude Code agents — each grounded by ENTERPRISE_BRAIN.md — into a pipeline interface for forensic expert witness report generation.

◆

Target Users

Lane Swainston — Owner/Stakeholder/Decision Maker. Reviews findings, approves stages, exports final reports. Needs zero-friction oversight.

Mariz — Power User/Operator. Uploads case documents, monitors agent progress, manages day-to-day case flow.

◇

Core Problem

Expert witness rebuttal reports require ingesting hundreds of pages, cross-referencing codes/standards, and producing legally defensible prose with full citation traceability. Currently manual — BlackBar's 4 agents automate this but have no UI.

○

Design Philosophy

Conversational flow, one CTA per screen, minimal color palette, generous whitespace. Designed for legal authority — dark mode, warm amber accent, glass morphism. Grit Score target: 1 (zero learning curve).

□

UAT Scope

Functional prototype for training Lane & Mariz on the workflow. Hardcoded auth (2 users). SQLite database. Local file storage. Express.js backend wrapping agent CLI calls. Not production-hardened — iterate first, harden later.

Design Principles

One CTA Per Screen

Every view has exactly one primary action. No competing buttons, no decision paralysis. Core UX principle — the user always knows what to do next.

Glass Morphism Cards

Semi-transparent surfaces with backdrop-blur, subtle border glow on hover. Creates depth without heaviness. Cards are the primary container for all content.

Pipeline-Driven Navigation

The 4-stage pipeline (Intake → Research → Draft → QA) is always visible. Users can see where they are, what's next, and what's done. Progress is the primary navigation.

Streaming Agent Activity

Real-time SSE feed showing what each agent is doing. Like watching a craftsman work — builds trust in the AI process. Right sidebar, always visible during active stages.

Page Architecture

8 views, one shell. Every screen maps to the pipeline.

01

Login

Centered card, dark background with subtle amber gradient glow. Email + password. Two users hardcoded (Lane, Mariz). Fraunces headline: "BlackBar". Single CTA: "Sign In".

02

Dashboard

Case overview. Stats bar (active cases, completed, avg turnaround, pending review). Case list with stage badges. "+ New Case" CTA. Right panel: recent agent activity across all cases.

03

Case Detail — Intake

Three-column layout. Left: case metadata sidebar. Center: document upload dropzone + case form — name (defendant adv plaintiff per Brain §5), case type (Brain §3 taxonomy dropdown), report type (Brain §4), jurisdiction, opposing expert, deadline. Right: agent activity feed. CTA: "Start Analysis".

04

Case Detail — Research

Findings displayed as citation cards with source traceability. Each card: claim → code reference → agent reasoning. Summary panel at top. Human checkpoint: "Approve Research" CTA.

05

Case Detail — Drafting

Rich text editor showing report draft. Section headers in sidebar for quick nav. Inline agent comments. Lane can edit directly. "Approve Draft" CTA at bottom. Revision request option.

06

Case Detail — QA

Automated check results: citation accuracy, voice match (Brain §5 prohibited/required terms scan), formatting compliance (Brain §12), benchmark test ("Does it read like Gleason, Heagy, or Anderson?" — Brain §11). Pass/fail per category. Issues list with severity. "Approve for Export" CTA or "Send Back" secondary.

07

Export

Final report preview (PDF render). Download options: PDF, DOCX. Metadata summary (case, expert, date, version). "Download Report" CTA. Clean — no editing here, just delivery.

08

Human Checkpoint Modal

Overlays at every stage transition. Summary of what the agent produced. Three options: Approve (advance), Revise (add notes + re-run), Reject (stop pipeline). Prevents runaway agent execution.

Enterprise Brain

ENTERPRISE_BRAIN.md — the domain knowledge layer that grounds every agent's Orient step in SCG's actual practice.

◆

What It Is

A structured knowledge document derived from Lane's VOICE.md (479 lines, 3 benchmark reports, 9+ filed expert reports). Contains everything the 4 agents need to sound like SCG — not generic AI.

◇

Why It Exists

Without the brain, the Drafting Agent would say "I visited the premises" instead of "SCG Personnel visited the subject premises." It would use "prior to" instead of "before." Every prohibited term, every voice rule, every attack pattern lives here.

○

How It Works

Each agent's OODA loop has an Orient step. During Orient, the agent queries specific brain sections for domain context. The brain returns grounded answers — not hallucinations.

□

Drift Detection

10 golden set questions (BB-001 through BB-010) validate the brain hasn't drifted from operational truth. LLM-as-judge scoring, 0–100 per question, flag <70. Run on every brain edit.

Voice Constraints (from Brain §5)

Entity Voice

✅ "SCG Personnel visited the subject premises..."
❌ "I visited the premises..."

The entity speaks, not the person. Two narrow exceptions: CBO credentials anchor and "(Lane)" parenthetical.

Prohibited Terms

❌ negligent, causation, fault, victim, safe/unsafe, prior to, clearly, obviously, dangerous condition, in my opinion

These are legal conclusions for counsel — not expert assertions.

Required Terms

✅ "Plaintiff Expert" (never "opposing expert")
✅ "allegedly" (always)
✅ "contributing factor(s)"
✅ "payment of attention"
✅ European date format

QA Benchmark Test

"Could this paragraph appear, word for word, in a filed expert report in Clark County District Court without a single edit? Does it read like Gleason, Heagy, or Anderson?"

If not, revise until it could.

Architecture Decisions

Every choice, rationale, and tradeoff — documented for Claude Code execution.

DecisionChoiceRationale
Frontend Framework Vite + React 18 + TS Claude Code iterates fastest with React. Vite gives instant HMR. TypeScript catches agent contract mismatches at build time.
Styling Tailwind + shadcn/ui Utility-first = Claude Code can style without stylesheet context switching. shadcn gives accessible primitives (modals, dropdowns, toasts) without vendor lock.
State Management Zustand 3 stores (auth, case, agent). Zustand is 1KB, zero boilerplate, and Claude Code generates it cleanly. Redux is overkill for 2 users.
Backend (UAT) Express.js + SQLite Monorepo with frontend. SQLite = zero infra. Express is the simplest thing Claude Code can scaffold. Swap to Postgres + proper API when going prod.
Agent Communication SSE (Server-Sent Events) Agents stream output. SSE is one-directional (server → client), perfect for activity feeds. Simpler than WebSockets. Native browser EventSource API.
Auth (UAT) JWT + hardcoded users Two users, no registration. Hardcode Lane + Mariz in db seed. JWT for stateless session. Swap to OAuth/SSO when going multi-tenant.
Rich Text Editor TipTap (ProseMirror) Lane needs to edit drafts. TipTap is extensible, has collaboration support for later, and Claude Code generates TipTap configs well.
File Storage (UAT) Local filesystem uploads/ directory. No S3 setup for UAT. Move to S3/R2 when going prod.
Display Font Fraunces Serif display font with optical size axis. Conveys authority + warmth — perfect for legal/forensic domain. Avoids the cold tech feel of geometric sans.
Accent Color #FF6B35 (Warm Amber) Warm amber signals precision + expertise without coldness. High contrast on dark backgrounds.
Deployment (UAT) localhost / LAN Lane and Mariz access via local network during training. No cloud deploy for UAT. Move to Vercel/Railway when validated.
Enterprise Brain ENTERPRISE_BRAIN.md Domain knowledge layer for agent Orient steps. 15 sections derived from VOICE.md. Drift-detected by golden set (BB-001–010). Lives in brain/ directory, loaded by agentRunner.ts with per-agent section routing.
